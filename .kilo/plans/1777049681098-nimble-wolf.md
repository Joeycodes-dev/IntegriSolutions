# ScanService.ts Review - South African License Card Support

## Current Implementation Summary

The scanning flow:
1. **Frontend** (`apps/mobile/src/services/scanService.ts`): Sends base64 image to backend API at `/api/scan`
2. **Backend Route** (`backend/src/routes/scan.ts`): Forwards to Gemini service
3. **Gemini Service** (`backend/src/geminiService.ts`): Uses Google Gemini AI to extract license data

## Assessment: Will it work for South African licenses?

**Partially - with critical issues to fix:**

### ✅ What's Working
- The prompt explicitly mentions "South African driver's license" (line 17 of geminiService.ts)
- Extracts core fields: name, license number, date of birth, expiry date
- Uses structured JSON output with schema validation

### ❌ Critical Issues

1. **Invalid Model Name** (`geminiService.ts:13`)
   - Current: `'gemini-3-flash-preview'` - this model does NOT exist
   - Valid options: `gemini-1.5-flash`, `gemini-1.5-pro`, `gemini-2.0-flash-exp`
   - **This will cause the scan to fail completely**

2. **Hardcoded Image MIME Type** (`geminiService.ts:18`)
   - Only accepts `image/jpeg`
   - Mobile scans may be PNG or other formats
   - Should detect/support multiple formats

3. **Missing South African Specific Fields**
   - SA licenses have ID number (13-digit), surname, initials, license codes
   - Current implementation only extracts basic fields
   - May need additional fields depending on use case

4. **Weak Error Handling**
   - `JSON.parse(response.text || '{}')` can throw on invalid JSON
   - No validation of returned data structure
   - No fallback if Gemini API fails

5. **Environment Dependency**
   - Requires `GEMINI_API_KEY` environment variable
   - No check if key is missing before making API call

## Recommended Fixes

1. Change model to valid Gemini model (e.g., `gemini-1.5-flash`)
2. Add MIME type detection or support multiple formats
3. Add proper error handling and response validation
4. Consider if additional SA license fields are needed (ID number, license codes)
5. Add API key validation on startup
